Description of the prospectus:¶

Why do gas station prices vary in different cities? Even more surprising is how prices differ in nearby cities.

As Energy Information Administration of the U.S. suggests that retail gas price is determined by crude oil price, refining cost, taxes, distribution costs, and gas station owner markup, the purpose of this project is to investigate and analyze how these variables affect retail prices.

By observing closely at the Northern California region, and especially nine big cities (San Francisco, Stockton, Santa Rosa, San Jose, Sacramento, Oakland, Modesto, Merced, and Santa Cruz), we rule out the effect from crude oil price, refining costs, taxes, and distribution cost since the costs are and should be the same for these nine adjacent cities.

Thus, the focus of this project has remained solely on the factors that influence station owner markup. By researching, the team has expected effects from local income, population density, automobile density, and cost of living to affect the ability of station owner to charge consumers.

Data Gathering:¶

Data gathering for this project has been rather troublesome and the team has experienced the most difficulty in this part. Unlike other data, which has been compiled nicely and should be conveniently downloaded, data for this project are hard to grep and compile.

Data 1 - City Gas Price: This data is downloaded from http://www.motortrend.com/gas_prices/31/california/. And the format is in HTML. Unlike normal XML and HTML file, there is not a node that contains exactly the price we want, and thus we can't easily fetch them through the XML and HTML module. Instead, we use regular expression from Bash to find the pattern of the price information. But another difficulty here is that the format of each line differs as there are cases when prices are not shown and are replaced by "N/A". Thus we have to adjust the regular expression to take these special cases in to account. After putting price information into txt file, we use python commands strip(), split(), and float() to drop the "

Data 2 - Income Data: During the process of gathering data part, I collect the income data. The income data comes from the website www.city-data.com. Some problems made the data not easy to collect. First of all, there isnt a complete data form, I need to search data city by city. Fortunately, most of these web page were set as the same form which let me easier to read and collect. I use bash and regular expression to read the HTML and clean the data, which lead a lot of task. The second problem is that I need to come up with a method to remove the comma in the number of income. After trying a lot of method, finally, I used python loops and conditions to complete the work and it works well.

Data 3 - Automobile density: In the process of gathering and cleaning data, my task is to obtain the number of automobiles in different counties from the official website of DMV. The original document is a PDF version of data source, and I converted this data source to txt format via Adobe acrobat Xi Pro. After that, I read my data into a data frame in Python. Since the data source shows a full list of numbers of automobile in California, my challenge is to retrieve some certain target counties that are included in our analysis and switch the data into the format that could be executed for later calculation. In order to solve that, I inserted a block of while and if code where I could locate the comma and avoid it.

Data 4 - Popualtion Density: I retrieved my data from www.usa.com. Since it is in html format, it is difficult to gather data using R. Therefore I used bash to retrieve the data, during which I confronted with certain challenges. First the command curl doesnt work in my notebook, so I used python to download the data, which is not so difficult. Second I reviewed regular expressions and used bash to grep those specific lines and then use sed to pipe all those data into a file. Third, by using Python, I managed to get rid of commas inside the data, and put all my data into dataframe.

Regression Refinement:¶

SECOND Challenge we encounter is to explain the high t-stat for population density and automobile density, and also take into account the fact that from the plot matrix we had, we see a clear and strong linear relationship between these two factors. (See the graph of population.density and automobile.density)

After researches, we conclude that there is a strong correlation between population density and automobile density and thus including both of them in the regression may incur double counting. Therefore we decided to drop the automobile density.

Also, we find that the relationship between income and gas price, population density and gas price should be an elasticity. It makes more sense to explain the relationship as "If one city's average income has increased 5%, the local gas price also increased a corresponding percentage). This means that we cannot simply use lm(gas.price ~ income+population.density+cost.of.living.index), instead we should modify the equation to be log(gas.price) ~ log(income) + log(population.density) + cost.of.living.index)

Conclusion:¶

From the results we get above, we conclude that there is a strong (but can still improve) relationship between gas price and income, population density, and cost of living. Interpretation of the regression results is that when a city's average income has increased by 1%, local gas price will raise 4.7%; when a city's population density has increased 1%, local gas price will drops 6.6%.
